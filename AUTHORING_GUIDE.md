# Authoring Guide

## Keep the skill narrow

One skill should do one job well. Avoid combining unrelated workflows.

## Preserve the 12-part anatomy

Each skill in this library uses:
- Purpose
- Use when
- Avoid when
- Inputs to gather
- Operating rules
- Antigravity tool pattern
- Expanded workflow
- Output contract
- Failure modes to avoid
- Handoff cues
- Example invocation
- Quality bar

## Description rules

Descriptions should say:
- what the skill does
- when it should trigger
- when it should not trigger

## Platform note

This pack is written for Antigravity. Keep repo-wide persistent guidance in `AGENT_NOTES.md` and keep reusable procedures in skills.
