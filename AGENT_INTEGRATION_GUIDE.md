# Agent Integration Guide

## What belongs where

### Put this in `AGENT_NOTES.md`
- architecture notes
- coding standards
- test commands
- review checklists
- non-negotiable repo rules

### Put this in skills
- repeatable workflows
- deep checklists
- multi-step operating playbooks
- specialized research or review procedures

## Routing habit

- Start with the smallest profile that covers the job
- Invoke a specialist explicitly the first few times
- Keep internal safety/orchestration skills installed only when their behavior is useful in practice
- Prefer explicit invocation for high-impact workflows

## Profile examples

- `minimal_core`: safe everyday baseline
- `builder_engineering`: debugging, review, migration, release, repo work
- `docs_support`: docs systems, guides, troubleshooting, support content
- `security_quality`: threat modeling, QA, release safety, appsec checks
